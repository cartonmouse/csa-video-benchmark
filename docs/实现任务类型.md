# EgoWorkSense 视频理解任务体系

## 一、Action Understanding (动作理解)
### 1.1 Basic Action Recognition (基础动作识别)
- 任务描述：识别视频片段中的单个动作
- 示例问题：工人对卷尺执行了什么操作？
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：verb + noun 标注字段

### 1.2 Fine-grained Action Recognition (细粒度动作识别)
- 任务描述：区分相似动作的细微差异
- 示例问题：工人是"拿起"还是"抓握"钢筋？
- 难度等级：Medium
- 实现状态：✓ 已完全实现
- 数据支持：拿起/抓握/搬运等细粒度verb标注

### 1.3 Multi-Action Recognition (多动作识别)
- 任务描述：识别一个视频中的所有动作
- 示例问题：视频中工人执行了哪些动作？(多选)
- 难度等级：Medium
- 实现状态：🟡 可快速实现
- 数据支持：多个segments标注

### 1.4 Action Antonym Recognition (动作反义识别)
- 任务描述：识别动作的反义操作
- 示例问题：与"拿起卷尺"相反的动作是什么？
- 选项示例：A.放下卷尺 B.移动卷尺 C.使用卷尺
- 难度等级：Easy
- 实现状态：🟡 可快速实现
- 数据支持：verb反义词库

### 1.5 Action Sequence Recognition (动作序列识别)
- 任务描述：判断动作的时间顺序
- 示例问题：以下动作的正确顺序是？
- 选项示例：A.拿起卷尺→移动钢筋 B.移动钢筋→拿起卷尺
- 难度等级：Medium
- 实现状态：✓ 已完全实现
- 数据支持：start_time + end_time 时序标注

### 1.6 Action Boundary Detection (动作边界检测)
- 任务描述：定位动作的起止时间
- 示例问题："拿起卷尺"动作发生在哪个时间段？
- 选项示例：A.0-1s B.0.9-2.5s C.3-4s
- 难度等级：Medium
- 实现状态：🟡 可快速实现
- 数据支持：精确的start_time/end_time标注

### 1.7 Action Duration Reasoning (动作时长推理)
- 任务描述：比较不同动作的持续时间
- 示例问题：以下哪个动作持续时间最长？
- 选项示例：A.拿起卷尺(1.5s) B.移动钢筋(10.1s)
- 难度等级：Easy
- 实现状态：🟡 可快速实现
- 数据支持：时间差计算

### 1.8 Concurrent Action Recognition (并发动作识别)
- 任务描述：识别同时发生的动作
- 示例问题：在1-3s内，发生了哪些动作？
- 难度等级：Hard
- 实现状态：🟡 可实现
- 数据支持：时间区间重叠分析

### 1.9 Action Frequency Analysis (动作频率分析)
- 任务描述：统计动作出现次数
- 示例问题：工人在视频中"拿起卷尺"几次？
- 选项示例：A.1次 B.2次 C.3次
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：相同verb+noun计数

## 二、Object Understanding (物体理解)
### 2.1 Object Recognition (物体识别)
- 任务描述：识别视频中出现的物体
- 示例问题：视频中出现了哪些物体？
- 选项示例：A.卷尺 B.钢筋 C.鞋子 D.模板
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：noun标注字段

### 2.2 Object-Action Binding (物体-动作绑定)
- 任务描述：识别动作和物体的配对关系
- 示例问题："拿起"动作作用于哪个物体？
- 选项示例：A.卷尺 B.钢筋 C.鞋子
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：verb + noun 配对

### 2.3 Object State Change (物体状态变化)
- 任务描述：推理物体状态变化
- 示例问题：执行"拿起卷尺"后，卷尺的状态变为？
- 选项示例：A.在地上 B.在手中 C.在工作台
- 难度等级：Medium
- 实现状态：🟡 可快速实现
- 数据支持：result字段描述

### 2.4 Object Existence Detection (物体存在性检测)
- 任务描述：判断特定时间段物体是否出现
- 示例问题：在5-10s期间，卷尺是否出现在画面中？
- 难度等级：Medium
- 实现状态：🟡 可实现
- 数据支持：noun + 时间区间分析

### 2.5 Object Interaction Pattern (物体交互模式)
- 任务描述：统计工人与物体的交互次数
- 示例问题：工人与钢筋交互了几次？
- 难度等级：Easy
- 实现状态：🟡 可实现
- 数据支持：相同noun出现频率统计

### 2.6 Object Co-occurrence (物体共现)
- 任务描述：识别同时出现的多个物体
- 示例问题：视频开始时，场景中有哪些物体？
- 难度等级：Medium
- 实现状态：🟡 可实现
- 数据支持：时间窗口内的noun集合

## 三、Temporal Reasoning (时序推理)
### 3.1 Immediate Predecessor (直接前驱)
- 任务描述：识别某动作的前一个动作
- 示例问题：在拿起卷尺之前，工人做了什么？
- 选项示例：A.放下鞋子 B.移动钢筋 C.无动作
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：时序segments排序

### 3.2 Immediate Successor (直接后继)
- 任务描述：识别某动作的下一个动作
- 示例问题：在放下鞋子之后，工人做了什么？
- 选项示例：A.拿起卷尺 B.移动钢筋 C.结束
- 难度等级：Easy
- 实现状态：✓ 已完全实现
- 数据支持：时序segments排序

### 3.3 Short-term Action Prediction (短期动作预测)
- 任务描述：预测接下来最可能的动作
- 示例问题：完成拿起钢筋后，工人接下来最可能做什么？
- 选项示例：A.绑扎钢筋 B.测量长度 C.清理现场
- 难度等级：Medium
- 实现状态：✓ 已完全实现
- 数据支持：next_action字段

### 3.4 Procedure-aware Prediction (流程感知预测)
- 任务描述：基于施工流程预测下一步
- 示例问题：完成"材料搬运"后，按照施工流程应该进行？
- 选项示例：A.工具准备 B.质量检测 C.材料安装
- 难度等级：Medium
- 实现状态：🟡 可快速实现
- 数据支持：procedure_type字段

### 3.5 Action Causality (动作因果)
- 任务描述：推理动作之间的因果关系
- 示例问题：工人为什么要先"拿起卷尺"再"移动钢筋"？
- 选项示例：A.需要测量钢筋长度 B.随机顺序
- 难度等级：Hard
- 实现状态：🟡 可实现
- 数据支持：动作语义推理

### 3.6 Temporal Gap Reasoning (时间间隔推理)
- 任务描述：分析两个动作之间的时间间隔
- 示例问题：从"拿起卷尺"到"移动钢筋"之间间隔多久？
- 选项示例：A.1-2s B.2-3s C.3-4s
- 难度等级：Easy
- 实现状态：🟡 可实现
- 数据支持：时间戳计算

### 3.7 Action Feasibility (动作可行性)
- 任务描述：判断动作的前置条件
- 示例问题：在什么情况下"测量长度"是不可行的？
- 选项示例：A.卷尺不在手中 B.光线太暗 C.钢筋已绑扎
- 难度等级：Medium
- 实现状态：✓ 已完全实现
- 数据支持：result + next_action逻辑推理

## 八、数据标注支持
### 核心标注字段
- start_time：动作起始时间
- end_time：动作结束时间
- verb：动作动词
- noun：交互物体
- actor：执行主体
- location：空间位置
- result：动作结果
- next_action：下一步动作
- procedure_type：流程类型

### 标注数据规模
- 总视频数：5个
- 已标注视频：4个有效
- 总segments：11个
- 平均每视频时长：15秒
- 动作类型覆盖：拿起、放下、移动、搬运、抓握

### 扩展性设计
- 支持多粒度标注：动作级、片段级、视频级
- 支持增量标注：新增字段不影响现有任务
- 支持跨模态扩展：可融合骨架、深度等数据
